java 多态

2018.01.06 Dfault0 java  热度 ℃

标签： java

概念

同一个对象在不同时刻所体现的不同状态
子类可以当做父类使用，但是父类不能当做子类使用（狗是动物，但动物不是狗）

多态的前提

  • 有继承或者实现
  • 有方法重写
  • 有父类或者父类解救引用指向子类对象 ： 父 F = new 子

多态的分类

1.具体类多态

2.抽象类多态

3.接口类多态

多态中的成员访问特点

1.成员变量
编译看father，运行看son
2.构造方法
创建子类对象的时候要先访问父类的构造方法，对父类的数据进行初始化（子类所有的构造方法都会先访问父类的无参构造）
3.成员方法
编译看father，运行看son（方法有重写）
4.静态方法
编译运行都是看father（和类相关）

多态的利与弊

1.好处
提高代码的维护性（继承）
提高代码的扩展性（多态）
2.弊处
父类不能使用子类的特有功能

多态中的转型

1.向上转型

2.向下转型：把父类的引用强制转换成子类的引用
