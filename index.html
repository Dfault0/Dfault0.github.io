git常用命令

  1. git init:初始化仓库
  2. git status : 查看当前仓库状态
  3. git add : 添加所有文件到仓库暂存区（修改的文件、新建的文件）
    • 参数：. 表示所有文件
    • 修改的文件：未暂存（unstaged）
    • 新建的文件：未跟踪（untracked）
    • 只添加部分文件：新建文件 .gitignore(不被git管理的文件)
  4. git commit -m “注释” ：提交暂存到仓库
  5. git log：查看仓库提交日志 （最开始显示的是最近的提交）
    • –all:查看全部
    • –pretty=oneline：显示一行 ，前面的标识短了一些
    • –abbrev-commit:优化显示
    • –graph:
    • –decorate：Mac 版本可能需要
  6. git reset –hard commitID
    • git reflog:查看已经删除的提交记录

分支

  1. git branch: 查看分支
  2. git branch branchName: new branch
  3. git checkout branchName :switch branch
  4. git checkout -b branchName: create and switch to branch
  5. git merge 分支名称：把分支名称的分支合并到当前分支
    • 存在冲突，如同时修改了同一个文件的同一行，就需要解决冲突：处理文件中冲突的地方，将文件添加到暂存区，提交到仓库即可
  6. 删除分支
    • git branch -d b1:删除分支时有相关检查
    • git branch -D b1: 不做任何检查，强制删除
  7. 分支代码：
    • master：生产分支，线上分支，主分支
    • develop：开发分支
    • hotfix：热修复
    • feature：新功能

远程仓库

  1. 添加全程仓库：
    • git init
    • git remote add origin <仓库路径 SSH>
    • git remote: 查看远程仓库
  2. git brand -vv: 查看远程分支和本地分支的关系
  3. 推送到远程仓库：
    • git push [-f] [–set-upstream] [远端名称 [本地分支名称] [:远端分支名称]]
    • -f :强制覆盖
    • –set-upstream:推送到远端的同时并且建立起和远端分支的关联关系。
      • git push –set-upstream origin master
    • 如果当前分支已经和远端分支关联，则可以省略分支名和远端名
      • git push

编译与使用OLLVM

编译OLLVM

  • 参考https://github.com/yangyiyu08/ollvm-project/tree/14.x
  • 以及之前的源码，需要添加一个入口点（entry.cpp）：主要作用是提供相关参数以及运行
  • 注意在源码外编译以及在源码内部编译都是可以的
  • 但是建议在源码外编译：方便调试（速度快）、以及不影响LLVM

使用OLLVM

  • clang使用：clang -Xclang -load -Xclang /home/fourth/ollvm/OLLVM/cmake-build-release/ollvm/lib/Transforms/Obfuscation/LLVMObfuscation.so -mllvm -fla -emit-llvm -S hello.c -o hello_fla.ll

混淆

  1. 如果存在ollvm混淆的话，就先去导出表里面看是否有.datadiv_decode（这个是OLLVM默认的字符串混淆函数），可以直接hook registerNative函数

  2. 如果没有的话就去看看.init_array里面有没有相关的函数，然后去看函数

  3. 同时也可以使用inlinehook，其方法和别的hooknative方法是一样的，只是如果不是函数的话，只能打印出当前的寄存器的值。

  4. 在注入的时候可能这个相关so还没有加载起来，所以找不到函数的地址，这个时候就需要hook dlopen,等到库加载起来的时候才进行inline hook（32位不太稳定，因为指令的长度不定，64位要好一点）

  5. 有些使用需要使用主动调用：为了固定住参数，在多次进行运行分析的时候会减轻工作量，同时可以在进行主动调用的时候需要多次执行，因为这样可以看出每一次的运行结果是否一致。

  6. 在控制流平坦化中，直接看代码是无意义的，可以只用看与关键参数相关的代码，X交叉引用，参数个数识别失败的，可以使用Y，直接修改函数的类型，一般是可以自动识别出来

  7. 看后面的参数是否继续用到了这个参数，如果后面没有用到这个参数就不看

  8. 在hook的时候，首先使用hexdump来打印出内容，然后再根据具体的内容判断数据类型，最后根据类型来打印具体的值，如果猜不到具体的值是什么的话，就先跳过，后面再看

  9. 在hook的时候如果，参数和返回值都没有变化的话，就说明这个函数没啥用，只是为了干扰，可以直接跳过

  10. 如果在分析一个函数的时候如果常量值较多的话，可以直接谷歌搜索该常量，就可以发现是否是一个特定的算法的常量值，进而推断出，该算法的主要目的

  11. 可以从输入分析，也可以从输出分析

  12. 在虚假控制流中的导出函数中，有很多x和y

  13. 注意在ARM64中，如果函数的返回值的内容大于16个字节，那么将使用X8来存储返回值的地址，所以Frida的返回值是没有生效的，需要自己直接获取X8的内容
