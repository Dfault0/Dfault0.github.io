编译llvm14

编译过程

在下载llvm源码的时候需要根据NDK的版本来选择:

  • android-ndk-r25b/toolchains/llvm/prebuilt/linux-x86_64/bin clang –version ->14.0.6

  1. 克隆llvm

  2. 使用Clion编译：

    为了使得clion可以直接用，并且节约空间，建议只使用这种方法来进行变异

    • 打开llvm/cmakelists.txt文件
    • setting->build,exe,dep->cmake->debue/release->camke options: -G Ninja -DCMAKE_BUILD_TYPE=Release -DLLVM_ENABLE_PROJECTS=’clang’
    • 文件导入结束之后：切换到llvm/cmake-build-debug/release文件夹：ninja -j8 / ninja clang编译

  3. build debug: 编译时间长

    • cd llvm -> mkdri build_llvm->cd build_llvm->cmake -G Ninja ../llvm->ninja -j8开始编译

  4. build release: 编译时间短

    • cmake -G Ninja -DCMAKE_BUILD_TYPE=Release -DLLVM_INCLUDE_TESTS=OFF -DLLVM_ENABLE_PROJECTS=”clang” ../llvm

编译出错

空间耗尽

  1. 扩展磁盘空间
  2. 将扩展的磁盘分配给现存的分区：disks->resize
  3. 增大交换空间大小：
    • sudo dd if=/dev/zero of=/swapfile1 bs=1G count=30 ：创建交换文件30G
    • free -h
    • sudo mkswap -f /swapfile1
    • sudo chmod 0600 /swapfile1
    • sudo swapoff /swapfile
    • sudo swapon /swapfile1
    • free -h
    • sudo vim /etc/fstab ->swapfile修改为swapfile1：设置开机自启动

使用llvm

  1. 将编译好的clang添加到环境变量：export PATH=/home/fourth/ollvm/llvm-project-llvmorg-14.0.6/llvm/cmake-build-release/bin:$PATH
  2. 使用clang编译相关文件：clang hello.c -o hello_clang->生成的文件可直接执行
  3. 调试clang：
    • 在开始打开的project中，在clang->tools->driver->driver.cpp->main()下断点
    • 在窗口build旁边选择clang|release（debug目前还没有编译完成）,edit configuration->clang->program arguments: /home/fourth/ollvm/study/hello.c -o /home/fourth/ollvm/study/hello_clion
    • 但是注意，调试还是需要debug版本的
  4. 使用llvm编译源文件流程：
    • .c->.ll:clang -emit-llvm -S hello.c -o hello.ll（ll文件可以直接修改后执行）
    • 执行ll文件：lli hello.ll
    • 将可读的LLVM转成bitcode：llvm-as hello.ll -o hello.bc
    • 将bitcode(字节码)文件转成本地汇编文件：llc hello.bc -o hello.s (llc后端编译器)
    • 使用clang将s转成可执行文件：clang hello.s -o hello_s
    • 将字节码文件反转成IR文件：llvm-dis hello.bc -o hello_re.ll

  3. 修改llvm/lib/Transforms/EncodeFunctionName/CMakeLists.txt

  4. 修改llvm/lib/Transforms/IPO/CMakeLists.txt：在最后加上EncodeFunctionName

  5. 修改llvm/lib/Transforms/IPO/PassManagerBuilder.cpp

  6. 这样是可以成功编译，但是llvm14上，使用clang -mllvm -encode-function-name hello.ll 是没有效果的，经过查看资料，应该是在12以下是可以起作用的

编写PASS llvm12以上

  1. 首先编写llvm/include/llvm/Transforms/EncodeFunctionName/EncodeFunctionName.h文件

  2. 编写llvm/lib/Transforms/EncodeFunctionName/EncodeFunctionName.cpp

  3. llvm/lib/Passes/PassRegistry.def中添加：FUNCTION_PASS(“encode-function-name”, EncodeFunctionName());

  4. llvm/lib/Passes/PassBuilder.cpp中添加：#include “llvm/Transforms/EncodeFunctionName/EncodeFunctionName.h”

  5. llvm/lib/Transforms/IPO/CMakeLists.txt的component添加：EncodeFunctionName

  6. 编译，运行：opt -disable-output /tmp/a.ll -passes=encode-function-name

  7. 编译，结果生效

使用pass

  1. 编译编写的pass
    • 可relase可debug
    • cd cmake-build-release
    • ninja EncodeFunctionName [编译完成之后会生成./lib/EncodeFunctionName .so]
  2. 使用opt使用pass:
    • opt -enable-new-pm=0 -load ‘/home/fourth/ollvm/llvm-project-llvmorg-14.0.6/llvm/cmake-build-release/lib/EncodeFunctionName.so’ -encode ./hello.bc -o hello.ll
    • 使用clang编译ll到可执行文件：clang hello.ll -o hello_encode
    • 最后生成的可执行文件就是经过了编写的pass的。
  3. 使用clang直接加载pass：
    • clang -Xclang -load -Xclang /home/fourth/ollvm/study/outPass/cmake-build-release/EncodeFunctionName/EncodeFunctionName2.so -encode hello.ll -o hello_encode_clang
