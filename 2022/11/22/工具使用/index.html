IDA调试APK

普通调试

  1. 将android_server push到手机上：#adb push android_server /data/local/tmp/
  2. 修改android_server权限：# chmod a+x android_server
  3. 启动android_server：#./android_server -p port
  4. 转发端口号：#adb forward tcp:23946 tcp:23946
  5. 设置调试选项:debugger-remote->debug ->host：127.0.0.1 port:23946
  6. 附加到进程

ida trace:主要思想，利用frida主动调用，利用IDA拦截调试

  1. 需要去掉Frida的注入：setImmediate(main)，然后使用Frida连接到应用程序（写好主动调用的函数）
  2. 使用IDA debug 加载应用程序，
  3. File->Script file->附件python脚本文件（脚本文件里面写好命令，断点之类）
  4. 使用Frida主动调用关键函数
  5. 运行IDA，就会停在断点处，然后挂起其他线程（防止其他线程里面有反调试）
  6. ida->debugger->tracing->tacing options->设置记录文件以及取消勾选“tarce over debugger sedments”
  7. ->instrucing tracing 开始tarce
  8. 得到trace的结果之后，需要结合frida的函数返回结果（结果可能在参数中），从结果分析具体的算法，主要是看寄存器的值，再写算法还原，在还原过程中需要多测试不同的参数，需要保证基本都正确才能说明算法还原正确，注意在调试过程中，需要根据条件来动态调试
  9. 在分析trace日志的过程中，使用010editor打开文件进行分析，方便

AS调试apk

  1. 确保手机的ro.debugger是开启的，如果没有开启则将其设置为1：

    • 使用magisk刷入MagiskHide
    • adb shell props: 根据提示修改或者添加 “Add/edit custom props”
    • 添加的属性包括：ro.debuggable=1 ro.secure=0

  2. AS直接导入apk：profile or debug apk

  3. 启动指定acticity(如下) or 启动app

    • adb shell

    • am start -D -n com.csd.newyunketang/.view.myLessons.activity.LocalVideoDetailActivity

  4. 需要设置端口转发：

    • adb shell ps | findstr com.csd
    • adb forward tcp:8900 jdwp:7456 （pid）
    • 这个8900端口需要在vs中用到

  5. AS设置断点，注意普通的smali support支持断点，所以需要下载新的smalidea插件，并且需要修改相关的设置

    • 重启AS，依次点击File -> Settting ->Editor -> File Types
    • 有两个smali名字的插件，一个是smali support ，一个是刚安装的smalidea
    • 将其原本的smali support支持的*.smali换到smalidea中
    • 设置断点

  6. AS设置Remote JVM debug

  7. attach remote android process:

    • 为了防止退出，所以需要debug type : java only

objection使用：

  1. Objection启动方式：

objection -N -h 10.16.60.128 -p 8888 -g com.bige0.shadowsocksr explore

objection -g com.bige0.shadowsocksr explore

  1. objection 常用指令
  • 查看内存中加载的库：memory list modules
  • 查看某个库的导出函数：memory list exports libssl.so
  • 在内存堆中搜索：android heap search instances com.android.settings.DisplaySettings
  • 执行：android heap execute handleID handleID中的某个函数名
  • 查看当前可用activity：android hooking list activities
  • 直接启动某个activity：android intent launch_activity com.android.settings.DisplaySettings。这个可以用来绕过某些activity
  • 查看包含关键字的类： android hooking list classes
  • 查看指定类里面的方法：android hooking list class_methods com.android.settings.DisplaySettings
  • 查看某个类的动态信息：android hooking watch class android.bluetooth.BluetoothDevice
  • 在找到相关的某些信息之后，堆设备进行相关的操作会得到打印出一些有用的信息
  • 动态查看某个类下面的某个函数的参数，返回值，调用栈：android hooking watch class_method android.bluetooth.BluetoothDevice.getName –dump-args –dump-return –dump-backtrace
  • 在hook单个函数的时候会HOOK所有的重载（函数名相同，参数不同）：android hooking watch class_method java.io.File.$init –dump-args
  • 加载插件：有两种方式，一种是在objection里面：plugin load 插件的具体位置；另一种是直接在使用objection的时候就加载插件：objection -N -h 192.168.3.131 -p 5555 -g com.android.settings explore -P ~/.objection/plugins
  • 加载插件wallbreaker ,查看某个类中的变量函数签名等信息：
    • plugin wallbreaker classdump –fullname android.bluetooth.BluetoothDevice

xposed 使用

基本步骤：

  1. copy XposedBridgeApi.jar到项目得libs目录，在app下面得build.gradle中添加依赖

    compileOnly files('libs/XposedBridgeApi-54.jar')

  2. 修改app目录下的build.gradle，并在AndroidManifest.xml文件中增加Xposed相关内容

  3. 新建Hook类，编写hook代码 必须是public得

  4. 在project模式下得main目录下新建assets文件夹，然后再assets目录下新建文件xposed_init，在里面写上hook类得完整路径，比如：com.example.xposed01.hookConstructors，否则会找不到

Linux

vim

三种模式：普通模式、插入模式、命令模式

  1. 普通模式->插入模式：
    • i 光标不动，在原光标的前面插入数据
    • a光标后移一位，在光标后面插入数据
    • I在本行的开始位置插入数据
    • A在本行的结束位置处插入数据
    • o 在本行下面创建一个空行
    • O 在本行的上一行创建一个空行
  2. 普通模式下的命令较多
    • dd删除当前行
  3. 命令模式下的命令：
    • wq 保存并退出文件
    • q 退出文件
    • q! 强制退出

查看文件

  1. cat [-n -b ] filename
  2. tac 倒叙查看文件内容 （从最后一行开始）
  3. head [-n] filename直接查看,默认前10行
  4. tail [-n] filename 倒叙查看，默认前10行
  5. more filename 文件过大，分页显示内容，空格建翻页
  6. wc 文件名 统计一个文件中各种数据的数量计算文件的byte 数，字数，列数等

linux实用小工具

  • jnettop：流量查看工具，可以查看下载和安装进度

使用工具生成dex文件

自己制作dex文件，方便再同样的Java环境下，对一些数据进行解码等操作

  1. 使用AS编写Java文件，build,在app\build\intermediates\javac\debug\classes\下找到相关的class文件

  2. D:\Android\Sdk\build-tools\29.0.3>d8 target.class

  3. adb push classes.dex /data/loal/tmp/

  4. chmod 777 /data/local/tmp/classes.dex

ida tips

  • 如果函数本来是有参数的是，但是ida没有识别出来：直接按Y即可
  • 如果是jni函数，则第一第二个参数固定：JNIEnv *env, jobject thiz
  • ctrl + F11然后右键，加入新的库之后（不需要加入头文件），就可以修改了
  • 如果函数没有识别出相关的参数，那么直接右键 force call type或者”Y”，修改函数声明即可
