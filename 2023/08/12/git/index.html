git常用命令

  1. git init:初始化仓库
  2. git status : 查看当前仓库状态
  3. git add : 添加所有文件到仓库暂存区（修改的文件、新建的文件）
    • 参数：. 表示所有文件
    • 修改的文件：未暂存（unstaged）
    • 新建的文件：未跟踪（untracked）
    • 只添加部分文件：新建文件 .gitignore(不被git管理的文件)
  4. git commit -m “注释” ：提交暂存到仓库
  5. git log：查看仓库提交日志 （最开始显示的是最近的提交）
    • –all:查看全部
    • –pretty=oneline：显示一行 ，前面的标识短了一些
    • –abbrev-commit:优化显示
    • –graph:
    • –decorate：Mac 版本可能需要
  6. git reset –hard commitID
    • git reflog:查看已经删除的提交记录

分支

  1. git branch: 查看分支
  2. git branch branchName: new branch
  3. git checkout branchName :switch branch
  4. git checkout -b branchName: create and switch to branch
  5. git merge 分支名称：把分支名称的分支合并到当前分支
    • 存在冲突，如同时修改了同一个文件的同一行，就需要解决冲突：处理文件中冲突的地方，将文件添加到暂存区，提交到仓库即可
  6. 删除分支
    • git branch -d b1:删除分支时有相关检查
    • git branch -D b1: 不做任何检查，强制删除
  7. 分支代码：
    • master：生产分支，线上分支，主分支
    • develop：开发分支
    • hotfix：热修复
    • feature：新功能

远程仓库

  1. 添加远程仓库：
    • git init
    • git remote add origin <仓库路径 SSH>
    • git remote: 查看远程仓库
  2. git brand -vv: 查看远程分支和本地分支的关系
  3. 推送到远程仓库：
    • git push [-f] [–set-upstream] [远端名称 [本地分支名称] [:远端分支名称]]
    • -f :强制覆盖
    • –set-upstream:推送到远端的同时并且建立起和远端分支的关联关系。
      • git push –set-upstream origin master
    • 如果当前分支已经和远端分支关联，则可以省略分支名和远端名
      • git push

克隆仓库

  1. git clone 仓库名
  2. 抓取指令：git fetch [remote name] [branch name]
    • 只是把仓库中 的更新抓取到本地，并不会合并
    • 需要手动merge: git merge origin/master
  3. 拉取指令：git pull [remote name] [branch name]
    • 直接合并
    • 相当于fetch + merge
